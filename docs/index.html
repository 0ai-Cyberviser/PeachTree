Live fuzzing signals into safe training datasets.

PeachTree is the dataset control plane for fuzz reports, minimized reproducers, test artifacts, and repository knowledge from assets owned or explicitly controlled by CyberViser / 0AI.

Build the pipeline Pair with PeachFuzz 
$ peachtree policy
owned_assets_only: true
public_scraping: disabled
review_required: true

$ peachtree ingest-local --repo ~/peachfuzz --repo-name peachfuzz --output data/raw/peachfuzz.jsonl
$ peachtree build --source data/raw/peachfuzz.jsonl --dataset data/datasets/peachfuzz-instruct.jsonl --manifest data/manifests/peachfuzz.json --domain peachfuzz
$ peachtree readiness --dataset data/datasets/peachfuzz-instruct.jsonl --fail-on-gate

02Review-first datasets

03No automatic training

Trainer handoff remains dry-run until an explicit human approval and external compute plan exists.

Pipeline

From live fuzzing to training-ready JSONL

Collect

PeachFuzz, CI reports, local regression tests, crash metadata, docs, and repo notes.

Normalize

Convert source artifacts into traceable PeachTree source records with stable IDs.

Gate

Block secrets, unknown licenses when disallowed, unsafe content, duplicates, and low-quality samples.

Export

Emit ChatML, Alpaca, or ShareGPT JSONL for Hancock training/evaluation experiments.

Live fuzzing

Use fuzzing as a dataset signal, not an unsafe autopilot.

PeachTree treats live fuzzing output as reviewed evidence: stack traces, minimized payloads, regression tests, corpus summaries, coverage notes, and fix explanations. It does not launch unauthorized fuzzing, network scanning, exploit execution, or third-party collection.

Allowed

Owned repos, local parsers, local corpora, CI-safe harnesses, synthetic payloads, minimized reproducers.

Blocked

Unowned targets, credential harvesting, raw secrets, private customer data, blind public scraping, destructive payloads.

Dataset lanes

Training data lanes for new Hancock models

Boundary

Assets you own only.

This site and pipeline are scoped to CyberViser / 0AI assets and explicit authorized repositories. Private credentials, raw telemetry, customer data, Google Admin URLs, Drive links, Proton Bridge secrets, and model provider keys stay out of public artifacts.
